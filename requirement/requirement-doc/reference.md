- **GB/T 8566-2007** — 软件生存周期过程

## 需求描述常用句式

### 功能需求描述
```
系统应/必须/应当/需要...【系统行为】
系统不应/不需要...【系统约束】
系统可以/能够...【系统能力】
```

### 性能需求描述
```
响应时间：... 操作应在 X 秒内完成
吞吐量：系统应支持 X 并发用户
容量：系统应存储 X 条记录
```

### 安全需求描述
```
系统应防止...【安全威胁】
系统应验证...【身份认证】
系统应授权...【访问控制】
```

## 需求评审 Checklist

### 需求评审会议准备
1. 提前发送评审材料（至少提前3天）
2. 准备评审检查表
3. 指定记录人员和主持人
4. 控制评审时间（每章节≤30分钟）

### 需求评审常见问题
| 问题类型 | 示例 | 严重程度 |
|----------|------|----------|
| 缺失需求 | 未考虑离线场景 | 高 |
| 模糊需求 | "系统应快速响应" | 高 |
| 矛盾需求 | 两个需求相互排斥 | 高 |
| 不可测试 | "用户体验良好" | 中 |
| 过度设计 | 需求超出范围 | 中 |
