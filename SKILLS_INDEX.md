## 🎓 学习路径推荐

### 初级开发者路径
适合刚入门的信息化建设人员：
1. [需求文档编写](./requirement/requirement-doc/) → 需求基础
2. [数据库实现](./coding/database-impl/) → 数据基础
3. [脚本工具开发](./coding/script-tools/) → 自动化入门

### 中级开发者路径
适合有一定经验的开发人员：
1. [业务流程分析](./analysis/business-process/) → 业务理解
2. [API接口开发](./coding/api-dev/) → 接口开发
3. [单元测试设计](./testing/unit-testing/) → 质量保障

### 高级开发者路径
适合资深架构师和技术负责人：
1. [总体架构设计](./architecture/system-design/) → 架构能力
2. [后端开发实战](./coding/backend-dev/) → 深度开发
3. [系统测试管理](./testing/system-testing/) → 全面质量

### 全栈开发者路径
适合全流程项目负责人：
需求 → 分析 → 架构 → 编码 → 测试 完整路径

## 💡 使用说明

### 如何使用技能
1. **选择技能** — 根据当前开发阶段选择对应技能
2. **阅读文档** — 查看 SKILL.md 了解核心内容
3. **实践操作** — 运行 scripts 中的示例代码
4. **参考学习** — 使用 reference.md 深入学习

### 技能文件结构
每个技能都包含标准文件：
- `SKILL.md` — 主要学习文档
- `forms.md` — 实践表单和检查清单
- `reference.md` — 技术参考文档
- `scripts/` — 示例代码文件
